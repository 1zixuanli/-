一、工作态度
--------------------------------------------------------------------------------
本人对待毕业设计态度端正，能够主动查阅资料、按指导教师意见修改方案；坚持每周
（或每两周）与导师沟通进度，对开题所定“SSM + Vue 前后端分离”的技术路线认真落
实。目前已完成前端主要功能开发与联调演示环境搭建，后续将集中精力完成服务端与
论文撰写。


二、已完成任务（对照开题报告中的功能与技术路线）
--------------------------------------------------------------------------------
1. 在指导教师指导下完成开题，明确研究目标：构建车票查询、余票校验与预订、订单
   管理以及管理端维护与统计等核心功能；确定采用 SSM 作为后端、Vue 作为前端的
   总体架构。

2. 完成与课题相关的文献与资料调研，梳理铁路/高铁客运售票业务特点及常见 B/S
   架构实现方式，为需求分析与系统设计提供依据。

3. 完成系统需求分析与概要设计：划分用户模块、车票查询模块、订票与订单模块、
   管理端模块；拟定 REST 风格接口及统一响应格式（如 code、message、data），
   为前后端协作与论文“系统设计”章节提供素材。

4. 完成数据库概念结构与逻辑结构设计（用户、车次/时刻、订单等实体及关联），
   并形成与开题一致的表结构设想，为 MyBatis 映射与后续建表做准备。

5. 完成前端开发环境与工程化配置：建立基于 Vite + Vue 3 的前端工程，集成
   Vue Router、Element Plus、Axios；配置 package.json、入口页面等，解决开发
   环境下的依赖安装与运行问题，能够稳定执行 npm install、npm run dev 进行演示。

6. 完成用户端主要页面与流程（当前以 Mock 数据模拟后端，用于验证交互与业务流）：
   用户注册与登录；首页按出发站、到达站、日期及票种、高铁筛选等进行查询；
   车次列表展示席别、票价与余票；进入确认订单页选择席别、填写乘车人并提交订单；
   “我的订单”中查看订单、模拟支付与取消订单；订单与余票规则在 Mock 层与本地
   持久化中体现，便于答辩演示。

7. 完成管理端主要界面（Mock）：数据概览、用户列表、全部订单查询与筛选、车次
   与各席别余票展示及“放票”调整等，与开题中管理端功能相呼应；通过角色区分
   管理员与普通用户的访问范围。

8. 已整理与后端对接的接口约定思路，后续将 Spring MVC Controller 按相同路径与
   数据格式返回，即可替换前端 Mock，降低联调成本。


三、待完成任务
--------------------------------------------------------------------------------
1. 在 MySQL 中完成建库、建表及必要索引与约束，编写 MyBatis Mapper 与 XML（或
   注解方式），实现用户、车次、票种/余票、订单等持久化操作。

2. 基于 SSM 搭建后端工程：完成 Spring 与 MyBatis 整合、Spring MVC 分层；
   实现注册登录、车票查询、余票校验、下单与订单状态变更、管理端维护与统计等
   接口，关键业务在 Service 层保证事务与数据一致性（如防超卖）。

3. 将前端各模块中 ENABLE_MOCK 关闭或改为请求真实后端，完成前后端联调与异常处理；
   视开题要求考虑缓存（如 Redis）对热点车次查询的优化说明或实现。

4. 开展功能测试、接口测试及必要的性能或界面兼容性说明，整理测试用例与结果，
   写入论文“系统实现与测试”部分。

5. 撰写并修改毕业论文：完成绪论、相关技术、需求分析、总体设计、详细设计、
   系统实现、测试、总结等章节；按学校规范统一格式、图表与参考文献引用。

6. 根据导师意见多轮修改定稿，完成查重与答辩 PPT、演示环境最终确认。


四、现存在的问题
--------------------------------------------------------------------------------
1. 后端 SSM 与数据库实现尚在推进中，订单与余票等核心业务最终以服务端事务与
   锁机制为准，目前前端 Mock 仅用于流程演示，与生产级一致性、并发场景仍有差距。

2. 论文正文篇幅与部分章节的深度仍需加强，尤其是与已实现系统对应的截图、用例
   与数据库设计说明需要随开发进度同步更新。

3. 开发环境与工具链（Node 版本、构建配置等）曾遇到依赖与代理类问题，虽已排查
   解决，但后续联调阶段仍可能暴露新的环境与端口配置问题，需预留调试时间。

4. 参考文献的广度与规范性仍需在定稿前统一核对。


五、拟采取的措施
--------------------------------------------------------------------------------
1. 制定后端开发优先级：先完成用户与车次查询接口，再实现订单下单、支付状态与
   取消逻辑，最后完善管理端统计与维护接口；每完成一模块即与前端联调一小块。

2. 维护简易接口清单（路径、方法、请求与响应示例），保证与前端 Axios 调用一致，
   减少返工。

3. 与指导教师保持固定沟通节奏，及时提交阶段性成果（代码仓库说明、数据库脚本、
   论文提纲与草稿），根据反馈调整功能范围与写作重点。

4. 论文写作与编码并行：每完成一个功能模块即补充对应章节文字与截图，避免后期
   集中补写造成时间紧张。

5. 预留至少一至两周用于全文润色、格式检测、查重修改与答辩演练。
